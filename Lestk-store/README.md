# لستك — موقع مُعاد هيكلته ومُحسّن الأداء

## الهيكل
```
index.html                 (~100 KB، مُصغّر)
assets/css/style.css       (~53 KB، مُصغّر)
assets/js/script.js        (~15 KB، مُصغّر)
assets/images/             (22 صورة WebP، إجمالي ~748 KB)
```

## ما الذي تغيّر
- **حجم index.html:** من 2.4 MB إلى ~100 KB.
- **الصور:** استُخرجت كل صور Base64 (كانت 2.24 MB داخل HTML) وحُوّلت إلى WebP في `assets/images/` بجودة بصرية محفوظة.
  - صور المنتجات: 8–44 KB لكل صورة (تحت سقف 150 KB).
  - الخلفيات (hero/service): 28–128 KB.
  - أيقونات الفئات: 2–8 KB (مع شفافية).
- **CSS وJS:** فُصلا إلى ملفين خارجيين ومُصغّرين.
- **Lazy Loading:** كل وسوم `<img>` (68) تحمل `loading="lazy"` و`decoding="async"`.
- **JavaScript:** يُحمّل عبر `defer` (غير معطّل للعرض). أُصلح قوس `}` زائد كان خطأً في CSS الأصلي.
- **Swiper:** أُزيل تماماً لأنه لم يكن مُستخدَماً (وفّر ~140 KB من JS المعطّل للعرض).
- **Font Awesome:** يُحمّل الآن بشكل غير معطّل للعرض (مع بديل `<noscript>`).

## ضمانات السلامة (تم التحقق منها آلياً)
- **JS:** تدفق الرموز مطابق 100% للأصل — لا تغيير في السلوك ولا أخطاء console جديدة.
- **CSS:** نفس المحددات والخصائص بالضبط — الهوية البصرية محفوظة.
- **المحتوى:** النص الظاهر مطابق تماماً (1240 كلمة) — لا قسم ولا منتج مفقود.

## ملاحظة واحدة
قسم بطاقات المنتجات يستخدم 9 صور خارجية من **Unsplash** (محسّنة عبر `?w=400&q=80` ومحمّلة كسولاً). لم يكن بالإمكان توطينها في بيئة العمل لعدم توفّر اتصال بالإنترنت. لتوطينها لاحقاً: نزّل كل رابط، حوّله إلى WebP، ضعه في `assets/images/`، واستبدل رابط Unsplash بالمسار المحلي.

## التشغيل
ارفع المجلد كما هو إلى أي استضافة ثابتة (Netlify / Vercel / GitHub Pages / استضافتك). فعّل ضغط gzip/brotli على الخادم لتقليل الحمولة الحرجة إلى ~40 KB.
